Problem

Three things break on big Solana swaps.

01 · Spread

Market orders pay slippage twice.

A single $10k+ swap on a Solana aggregator hits AMM curves at full size — you eat the impact and the price-improvement that an RFQ pool would have offered.

02 · MEV

Sandwich attacks at the route level.

Public-mempool routes are visible to bots before they land. Your fill ends up in a sandwich, between a frontrun and a backrun, before the block confirms.

03 · Idle capital

The waiting half earns nothing.

Smart traders TWAP large positions over an hour. The portion still waiting to swap sits in their wallet — interest-free — while it could be earning Kamino yield.

Aggregators solve one of these. LIMINAL solves all three.

Solution

One terminal. Four partners. Composite alpha.

DFlow

RFQ-routed slices

Every slice clears against committed market-maker inventory in Jupiter Ultra's pool. Spread captured, MEV neutralised.

Kamino

Capital not yet swapped sits in the highest-APY Kamino lending vault, accrues per second, gets withdrawn 0.4s before each slice fires.

QuickNode

Live nervous system

Pyth feeds + slot-confirmation streams + dedicated RPC drive the price monitor that decides when each slice is allowed to execute.

Solflare

One signing surface

Wallet-native flow with versioned-transaction batching brings a 4-slice run from 10 popups down to 6, with simulation on every broadcast.

LIMINAL
Integration depth

Each partner is in the critical path, not bolted on.

DFlow Execution layer
  • Quote comparison on every slice — market vs. DFlow side-by-side, rendered live
  • MEV-protected execution via Jupiter Ultra RFQ + DFlow-endorsed market makers
  • Adaptive route fallback chain on MTU edge cases (excludes Whirlpool/Phoenix when needed)
  • Slippage threshold: defer-not-fail, single retry, then user prompt
Kamino Yield layer
  • Auto-vault selection by APY × liquidity × audit status, per input mint
  • Deposit on configure → partial withdraw 0.4s before each slice → final withdraw at completion
  • kToken accounting reconciles real yield earned, not estimated
  • Lending-only — no CLMM exposure, no impermanent loss
QuickNode Data layer
  • Dedicated RPC + Streams for slot confirmation under 500ms
  • Pyth price feed driving 5s polling loop on every active mint
  • Wallet event subscriptions for balance + kToken position deltas
  • Network meter in the header reads live RPC latency, drops bars when congested
Solflare UX layer
  • Wallet-adapter sole-supported on purpose — no Phantom fallback dilutes depth
  • Versioned transaction batching cuts 4-slice popups from 10 to 6
  • simulateTransaction before every broadcast — no silent failures
  • Deep linking + in-app browser auto-connect on mount
